# SuperSwingHelper Roadmap Checklist

이 문서는 현재 코드베이스를 기준으로 **탭 구조 정리 + 실시간 그래프(차트) 기능 추가**를 단계적으로 진행하기 위한 체크리스트입니다.

핵심 원칙:
- 대상 사용자는 **초보 부업 스윙 투자자**
- 자동매매 앱이 아니라 **시그널 알림기**
- 전략 빌더가 아니라 **쉽게 이해 가능한 신호 앱**
- 기능을 한 번에 늘리기보다, **구조 정리 → 데이터 계층 추가 → 시각화 → 실연동** 순서로 진행

---

## Phase 1. UI 정보구조 정리

목표:
- 한 페이지에 몰린 기능을 탭 구조로 분리
- 차트 기능이 들어와도 복잡하지 않게 만들기

추천 탭:
- 개요
- 워치리스트
- 차트
- 시그널
- 설정

### 체크리스트
- [ ] `page.tsx`를 탭 기반 레이아웃으로 분리
- [ ] `AppTabs` 컴포넌트 추가
- [ ] 개요 탭 생성
- [ ] 워치리스트 탭 생성
- [ ] 시그널 탭 생성
- [ ] 설정 탭 생성
- [ ] 차트 탭 placeholder 생성
- [ ] 기존 기능을 각 탭으로 이동
- [ ] 기능은 바꾸지 말고 구조만 먼저 정리

### 완료 기준
- [ ] 기존 기능이 모두 유지됨
- [ ] 차트 없이도 탭 전환이 가능함
- [ ] 초보자 기준으로 화면이 덜 복잡해짐

---

## Phase 2. 프론트 컴포넌트 리팩터링

목표:
- 탭 구조를 뒷받침할 수 있도록 기능 단위 컴포넌트 분리

### 추천 구조
- `components/layout/AppTabs.tsx`
- `components/dashboard/SummarySection.tsx`
- `components/watchlist/WatchlistSection.tsx`
- `components/signals/SignalsSection.tsx`
- `components/settings/SettingsSection.tsx`
- `components/chart/ChartSection.tsx`

### 체크리스트
- [ ] 현재 `page.tsx`의 상태 로직 정리
- [ ] 탭별 섹션 컴포넌트 분리
- [ ] 공통 타입은 `lib/types.ts`에 정리
- [ ] 공통 API 호출은 기존 `lib/api.ts` 유지
- [ ] UI 문구는 초보자 친화적으로 유지

### 완료 기준
- [ ] `page.tsx`가 오케스트레이션 역할만 함
- [ ] 차트 기능 추가 시 수정 범위가 국소화됨

---

## Phase 3. 차트 데이터 계층 추가

목표:
- 프론트가 차트를 그릴 수 있도록 차트 전용 REST 데이터 제공

### 추천 백엔드 API
- `GET /api/chart/{symbol}?limit=240`

### 응답에 포함할 것
- `candles`: 최근 1분봉 OHLCV
- `overlays`:
  - `ma20`
  - `ma60`
  - `bollinger_upper`
  - `bollinger_mid`
  - `bollinger_lower`
  - `rsi14`
- `markers`:
  - 최근 시그널 위치

### 체크리스트
- [ ] `ChartCandle` 응답 스키마 정의
- [ ] `ChartOverlayPoint` 스키마 정의
- [ ] `GET /api/chart/{symbol}` 엔드포인트 추가
- [ ] `CandleAggregator.get_recent_candles()` 활용
- [ ] 지표 계산을 차트 응답용으로 재사용
- [ ] 심볼별 최근 시그널도 같이 내려주기
- [ ] `limit` 기본값 지정

### 완료 기준
- [ ] mock 실행 중 특정 심볼의 최근 캔들 배열을 REST로 받을 수 있음
- [ ] 프론트 없이 curl/API 테스트 가능

---

## Phase 4. 실시간 차트 업데이트 이벤트 추가

목표:
- 초기 차트 데이터 외에, 실시간 차트 업데이트를 WebSocket으로 반영

### 추천 이벤트
- `candle_update`
- `candle_closed`

### 체크리스트
- [ ] WS 메시지 타입 확장
- [ ] runtime에서 봉 갱신 시점 이벤트 발행
- [ ] 심볼 기준 필터링 가능하게 설계
- [ ] 차트용 payload 최소화
- [ ] 기존 `signal`, `live` 이벤트와 충돌 없게 유지

### 완료 기준
- [ ] 프론트가 REST 초기 데이터 + WS 업데이트로 차트를 유지 가능
- [ ] 새 봉이 생길 때 append 가능

옵션 토글:

- [ ] `ChartSection.tsx` 구현
- [ ] REST 초기 데이터 로드

- [ ] RSI 토글 구현
- [ ] 최근 시그널 마커 표시

### 완료 기준

## Phase 6. 워치리스트와 차트 연결

목표:
- 워치리스트에서 본 종목을 바로 차트로 검토할 수 있는 흐름 만들기

### 원하는 UX
- 워치리스트 종목 클릭
- 차트 탭으로 이동
- 해당 종목 차트 자동 선택

### 체크리스트
- [ ] 선택된 심볼 전역 상태 추가
- [ ] 워치리스트 행 클릭 핸들러 추가
- [ ] 차트 탭 진입 시 선택 종목 반영
- [ ] 선택 종목이 없으면 첫 번째 감시 종목 기본 선택
- [ ] 최근 시그널 종목 클릭 시 차트 열기

### 완료 기준
- [ ] 워치리스트 → 차트 흐름이 자연스러움
- [ ] 알림 확인 후 바로 차트 검토 가능

3. 차트 탭 placeholder
