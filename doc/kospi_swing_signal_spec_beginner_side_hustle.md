# 코스피 스윙 시그널 알림기 스펙 문서

## 문서 정보
- 프로젝트명: `kospi-swing-signal`
- 문서 버전: v0.1
- 문서 목적: Codex 기반 개발 시작을 위한 MVP 중심 제품/기술 스펙 정의
- 대상 사용자: **초보 부업 스윙 투자자**
- 핵심 원칙: **쉽게 이해되고, 과도하게 복잡하지 않으며, 실시간으로 유용한 시그널을 주는 앱**

---

## 1. 제품 개요

### 1.1 한 줄 정의
코스피 종목을 실시간 감시하면서, **추세 + 거래량 + 지지/저항 + 볼린저밴드**를 기반으로 초보 스윙 투자자에게 **매수 후보 / 돌파 / 매도 경고** 알림을 제공하는 앱.

### 1.2 제품 목표
이 앱은 자동매매 툴이 아니라, 다음을 돕는 **실시간 시그널 알림기**입니다.

1. 초보 투자자가 차트를 계속 보고 있지 않아도 되게 만든다.
2. 스윙 매매에서 자주 쓰는 기본 논리를 이해하기 쉽게 제공한다.
3. 복잡한 HTS/MTS 대신, 필요한 정보만 간단하게 보여준다.
4. 부업 투자자 기준으로 **장중 계속 붙어있지 않아도 되는 경험**을 제공한다.

### 1.3 대상 사용자 정의
이 앱의 핵심 사용자는 다음과 같습니다.

- 직장인 / 학생 / 본업이 따로 있는 부업 투자자
- 한국 주식, 특히 코스피 중심으로 스윙 매매를 공부하는 사람
- 유튜브, 카톡방, 커뮤니티에서 용어는 보지만 실전 판단은 어려운 사람
- 자동매매보다는 **“알림을 보고 직접 판단”** 하는 방식을 선호하는 사람
- 복잡한 조건검색식을 직접 짜기 어려운 사람

### 1.4 사용자 페르소나
#### 페르소나 A: 초보 직장인 투자자
- 평일 장중에 차트를 계속 볼 수 없음
- 관심 종목은 몇 개 있지만 타이밍 판단이 어려움
- “볼린저 중앙선 지지”, “거래량 증가”, “돌파” 같은 말을 들어봤으나 조합이 어려움
- 복잡한 전략 대신 **간단한 강/중/약 알림**을 원함

#### 페르소나 B: 주식 공부 중인 스윙 입문자
- RSI, 볼린저밴드, 외인 수급 같은 용어는 조금 앎
- 실제로 어떤 시점에 의미가 있는지 헷갈림
- 단순한 가격 알림보다 **이유가 함께 적힌 시그널 알림**을 원함

## 2. 제품 원칙

### 2.1 초보자 친화적이어야 한다
- 용어를 그대로 보여주되, 항상 짧은 설명과 함께 제공한다.
- 전략을 자유롭게 코딩하게 하지 않는다.
- 복잡한 파라미터는 최소화한다.

### 2.2 자동매매가 아닌 의사결정 보조여야 한다
- 이 앱은 사용자의 판단을 돕는다.
- 매수/매도 버튼이나 주문 연동은 MVP에서 제외한다.
- 모든 시그널은 “추천”이 아니라 “조건 충족 알림”으로 표현한다.

### 2.3 과잉 알림보다 신뢰도 높은 알림이 우선이다
- 알림이 너무 많으면 초보 사용자는 곧 꺼버린다.
- 따라서 첫 버전은 **강한 신호 위주**로 제한한다.
- 1분봉 기반 판단과 중복 알림 방지 로직을 둔다.

### 2.4 한국식 스윙 문법에 맞아야 한다
- 추세
- 거래량
- 지지/저항
- 볼린저밴드
- 선택적 RSI

이 4~5개를 중심으로 구성한다.

### 2.5 학습 가능한 UX여야 한다
- “왜 이 알림이 떴는지”가 반드시 설명되어야 한다.
- 예: `볼린저 중앙선 지지 + 거래량 1.8배 + 전고점 돌파 시도`

## 3. 문제 정의

### 3.1 사용자가 현재 겪는 문제
1. 증권앱은 정보가 너무 많고, 초보자에게는 어디를 봐야 할지 어렵다.
2. 가격 알림만으로는 의미 있는 매수/매도 타이밍을 알기 어렵다.
3. 볼린저, 거래량, 지지/저항, 추세를 종합적으로 보는 것이 어렵다.
4. 장중 계속 차트를 볼 수 없는 사람에게 적합한 간단한 시그널 툴이 부족하다.
5. 커뮤니티에서 쓰는 용어를 이해해도, 그걸 실시간 판단으로 연결하기 어렵다.

### 3.2 이 앱이 해결하는 방식
- 관심 종목을 감시한다.
- 실시간 데이터를 받아 내부에서 1분봉을 만든다.
- 핵심 지표를 계산한다.
- 조건 충족 시, 이유가 적힌 시그널 알림을 보낸다.
- 로그를 남겨 복습할 수 있게 한다.

## 4. MVP 범위

### 4.1 포함 기능
#### A. 관심 종목 워치리스트
- 종목 추가/삭제
- 종목 그룹 관리
- 감시 ON/OFF

#### B. 실시간 시세 수신
- 종목별 체결가 수신
- 거래량 수신
- 장 상태 확인

#### C. 1분봉 생성
- 실시간 체결 데이터를 1분봉(OHLCV)으로 집계
- 최근 N개 봉 유지

#### D. 지표 계산
- 단기 이동평균선 (기본 20)
- 장기 이동평균선 (기본 60)
- 볼린저밴드 (기본 20, 2)
- 평균 거래량
- 전고점 / 전저점
- 선택적 RSI(14)

#### E. 시그널 생성
- 매수 후보
- 돌파 신호
- 매도 경고

#### F. 알림 발송
- 앱 내 알림
- 데스크톱 알림
- 텔레그램 알림

#### G. 시그널 로그
- 발생 시각
- 종목
- 신호 종류
- 발생 사유
- 당시 가격/거래량/주요 지표

#### H. 초보자용 설명 UI
- 용어 툴팁
- 각 시그널에 대한 짧은 설명

### 4.2 제외 기능
다음은 MVP에서 제외합니다.

- 자동매매
- 주문 연동
- 백테스트 엔진
- AI 예측 모델
- 모바일 네이티브 앱
- 고급 전략 빌더 (if/and/or 자유 설계)
- 커뮤니티 / 소셜 기능
- 포트폴리오 손익 자동 추적

## 5. 사용자 커스터마이징 범위

### 5.1 원칙
MVP에서는 **전략을 사용자가 직접 설계하게 하지 않고**, 아래와 같이 **정해진 전략의 파라미터만 조정**하도록 한다.

> 전략 로직은 고정, 파라미터는 사용자 조정

### 5.2 사용자가 조정 가능한 항목
#### A. 워치리스트
- 감시 종목 추가/삭제
- 종목 그룹 생성

#### B. 지표 파라미터
- 단기 MA: 기본 20
- 장기 MA: 기본 60
- 볼린저 length: 기본 20
- 볼린저 std: 기본 2
- 거래량 배수: 기본 1.5
- RSI 사용 여부

#### C. 시그널 조건 ON/OFF
- 볼린저 중앙선 지지 사용
- 거래량 급증 사용
- 전고점 돌파 사용
- RSI 교차 사용
- 추세 필터 사용

#### D. 알림 강도
- 모든 시그널
- 강한 시그널만

#### E. 알림 채널
- 앱 내
- 데스크톱
- 텔레그램

### 5.3 사용자가 조정하면 안 되는 항목
- 자유 전략 빌더
- 수식 직접 입력
- 복잡한 다중 조건 nesting
- 자동 주문 로직

## 6. 핵심 시그널 정의

### 6.1 공통 개념
시그널은 “예측”이 아니라 **정해진 조건이 충족되었음을 알려주는 이벤트**입니다.

각 시그널은 다음 정보를 가져야 합니다.
- 신호 종류
- 종목명 / 종목코드
- 발생 시각
- 현재가
- 충족된 조건 요약
- 간단 설명

### 6.2 시그널 종류
#### A. 매수 후보 (Buy Candidate)
목적: 본격 돌파 전, 관심 있게 볼 만한 자리 알림

기본 조건 예시:
- 단기 MA > 장기 MA
- 현재가가 볼린저 중앙선 부근에서 지지 확인
- 현재 거래량이 평균 거래량 대비 X배 이상
- 장중 급락/급등 노이즈 제외

알림 예시:
> 풍산: 매수 후보 신호. 볼린저 중앙선 지지, 거래량 1.7배, 20MA 상방 유지.

#### B. 돌파 신호 (Breakout)
목적: 저항 돌파 시점 감지

기본 조건 예시:
- 전고점 또는 최근 저항선 돌파
- 거래량 평균 대비 증가
- 추세 필터 통과

알림 예시:
> 삼성전자: 돌파 신호. 최근 전고점 돌파, 거래량 2.1배.

#### C. 매도 경고 (Sell Warning)
목적: 보유 종목이 약해지는 구간 경고

기본 조건 예시:
- 볼린저 중앙선 이탈
- 주요 지지선 이탈
- 거래량을 동반한 하락
- 장중 기준치 하향 이탈

알림 예시:
> 현대차: 매도 경고. 중앙선 이탈 및 거래량 증가 동반 하락.

### 6.3 신호 강도
- `weak`: 참고용
- `medium`: 주의 깊게 볼 만함
- `strong`: 즉시 확인할 가치 있음

강도 산정은 조건 충족 수와 핵심 조건 포함 여부로 결정한다.

## 7. 지표 및 판단 로직

### 7.1 추세
기본 정의:
- `MA_short > MA_long` 이면 상승 추세

기본값:
- `MA_short = 20`
- `MA_long = 60`

### 7.2 거래량
기본 정의:
- 현재 1분봉 거래량이 최근 N봉 평균 거래량 대비 X배 이상이면 증가로 간주

기본값:
- `N = 20`
- `X = 1.5`

### 7.3 지지/저항
MVP에서는 복잡한 매물대 분석 대신 다음만 사용한다.
- 최근 N봉 고점 = 단순 저항
- 최근 N봉 저점 = 단순 지지
- 볼린저 중앙선 = 동적 지지/저항 보조선
- 단기 이동평균선 = 동적 지지 판단 보조

### 7.4 볼린저밴드
기본 설정:
- `length = 20`
- `std = 2`

주요 해석:
- 중앙선 지지 = 추세 유지 가능성
- 상단 돌파 = 강한 상승 가능성
- 하단 이탈 = 약세 가능성

### 7.5 RSI (선택)
기본 설정:
- `period = 14`

MVP에서는 보조 조건으로만 사용한다.
- RSI가 시그널선 상향 돌파 시 가점

### 7.6 중복 알림 방지
동일 종목, 동일 신호, 동일 방향에 대해
- 일정 시간 내 중복 알림 금지

기본값 예시:
- `cooldown = 10분`

## 8. UX / UI 요구사항

### 8.1 핵심 UX 목표
- 초보자도 3초 안에 현재 상황을 이해할 수 있어야 한다.
- 이유 없는 숫자 나열 대신, **설명형 알림**을 제공한다.
- 차트를 몰라도 최소한 신호와 이유는 이해할 수 있어야 한다.

### 8.2 핵심 화면
#### A. 대시보드
표시 항목:
- 장 상태
- 오늘의 알림 수
- 강한 신호 종목 목록
- 워치리스트 요약

#### B. 워치리스트 화면
표시 항목:
- 종목명
- 현재가
- 변동률
- 최근 신호
- 감시 상태

#### C. 시그널 로그 화면
표시 항목:
- 발생 시각
- 종목
- 신호 종류
- 신호 강도
- 설명

#### D. 설정 화면
표시 항목:
- 지표 파라미터
- 조건 ON/OFF
- 알림 채널
- 알림 강도

### 8.3 UX 원칙
- 용어 옆 툴팁 제공
- “왜 떴는지” 문장형 요약 제공
- 초보자 모드에서는 어려운 수치보다 결과 요약 우선 노출
- 강한 신호는 시각적으로 더 강조

## 10. 데이터 소스 및 API 전략

### 10.1 추천 API
MVP 기준 1순위는 **한국투자증권 Open API**.

이유:
- 국내주식 실시간 데이터 접근 가능
- 웹소켓 기반 실시간 파이프라인 구성 가능
- 시그널 알림기용 실시간 감시에 적합

### 10.2 사용 방식
- 실시간 체결 데이터를 수신한다.
- 내부에서 1초 단위 버퍼링 후 1분봉을 생성한다.
- 봉 마감 시점 또는 이벤트 발생 시 조건을 평가한다.

### 10.3 틱레이트 / 평가 주기
권장 구조:
- 수신: 실시간 웹소켓
- 내부 버퍼: 1초
- 판단 기준: 1분봉

이유:
- 초보 스윙 투자자용 앱에서 틱 단위 판단은 노이즈가 많다.
- 1분봉이 속도와 안정성의 균형이 좋다.

## 11. 시스템 아키텍처

### 11.1 개요
```text
브로커 API (실시간 체결/거래량)
→ Data Ingestor
→ Candle Aggregator (1분봉 생성)
→ Indicator Engine
→ Signal Engine
→ Notification Service
→ Web Dashboard / Logs / Settings
```

### 11.2 컴포넌트 설명
#### A. Data Ingestor
- 브로커 API 연결
- 종목별 실시간 체결 데이터 수신
- 연결 끊김 시 재연결

#### B. Candle Aggregator
- 실시간 체결 데이터를 1분봉으로 집계
- OHLCV 계산
- 최근 N개 봉 관리

#### C. Indicator Engine
- 이동평균선 계산
- 볼린저 계산
- 거래량 평균 계산
- RSI 계산
- 최근 고점/저점 계산

#### D. Signal Engine
- 조건 평가
- 신호 강도 산출
- 중복 알림 억제

#### E. Notification Service
- 앱 내 알림
- 텔레그램 알림
- 데스크톱 알림

#### F. Dashboard API
- 프론트엔드에 현재 상태 제공
- 워치리스트, 로그, 설정 조회/수정

## 15. 초보자 보호 장치

이 제품은 초보 부업 스윙 투자자를 위한 앱이므로, 아래 장치를 기본 제공한다.

### 15.1 과도한 확신 표현 금지
금지 표현 예:
- 지금 반드시 사야 함
- 무조건 오른다
- 확정 수익 구간

권장 표현 예:
- 매수 후보 신호
- 조건 충족 알림
- 돌파 시도 감지
- 추세 약화 경고

### 15.2 리스크 안내
- 앱 시작 시 “투자 판단과 책임은 사용자 본인에게 있음” 고지
- 모든 시그널 화면에 작은 리스크 문구 표시

### 15.3 과잉 알림 방지
- 기본은 `강한 신호만`
- 동일 종목 중복 알림 제한

### 15.4 설명 중심 인터페이스
- 숫자만 던지지 않고, 이유를 문장으로 설명

## 20. Codex 구현 가이드

Codex는 아래 원칙을 따라 구현한다.

1. MVP 범위를 넘지 않는다.
2. 자동매매 기능을 넣지 않는다.
3. 초보 사용자를 위한 단순한 UX를 우선한다.
4. 전략 빌더 대신 프리셋 + 체크박스 + 숫자 파라미터 중심으로 구현한다.
5. 모든 시그널은 발생 이유를 사람이 읽을 수 있는 문장으로 생성한다.
6. 백엔드는 FastAPI, 프론트는 Next.js + TypeScript 기준으로 설계한다.
7. 시그널 엔진은 테스트 가능하게 분리된 모듈 구조로 작성한다.
8. 실시간 수신과 UI 렌더링을 느슨하게 결합한다.
9. 브로커 API 의존 코드는 별도 adapter 레이어로 분리한다.
10. 초보자가 오해할 수 있는 과장 문구를 금지한다.

## 23. 요약

이 앱은 **초보 부업 스윙 투자자**를 위한, 설명형 실시간 시그널 알림기입니다.

핵심은 다음과 같습니다.
- 자동매매가 아니다.
- 어려운 전략 빌더가 아니다.
- 한국식 스윙 매매의 핵심 요소를 간단하게 묶은 도구다.
- 장중 계속 차트를 볼 수 없는 사용자에게 유용해야 한다.
- “왜 이 신호가 떴는지”를 반드시 설명해야 한다.

MVP의 성공 기준은 기능이 많음이 아니라,
**적은 기능으로도 실제로 계속 켜두고 쓸 수 있는가**입니다.
